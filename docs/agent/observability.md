# Observability — Logging, Tracing & Swagger

Parent index: [AGENTS.md](../../AGENTS.md)

Use this doc to debug ingestion/query issues without scanning source.

---

## Quick debug workflow

1. **Reproduce** via [Swagger UI](#swagger--openapi) or `curl`.
2. **Poll status** — `GET /api/confluence/ingest/status/{parentPageId}` (no PAT).
3. **Read progress files** on disk — `crawl-progress.json`, `batch-progress.json`.
4. **Search logs** — `logs/app.log`; correlate by `traceId` in the log line.
5. **Check manifest** — `data/{parentPageId}/manifest.json` → per-page `lastError`, `noOfRetries`.
6. **Raise log level** to `DEBUG` for the failing package (see [Logging](#logging)).
7. **View traces** in an OTLP backend if a collector is running (see [Tracing](#distributed-tracing)).

### Documented endpoints

| Method | Path | Controller | Notes |
|--------|------|------------|-------|
| `GET` | `/health` | `IngestionController` | Simple `{"status":"ok"}` — not in actuator |
| `POST` | `/api/confluence/ingest` | `IngestionController` | Body: `IngestionRequest` |

Validation errors return **400** `ProblemDetail` via `GlobalExceptionHandler` (field-level messages).

### Try ingest in Swagger

1. Open Swagger UI → **POST** `/api/confluence/ingest` → **Try it out**.
2. Example body (init manifest only):

```json
{
  "baseUrl": "https://confluence.example.com",
  "parentPageId": "12345",
  "pat": "YOUR_PAT",
  "forceRebuildManifest": false,
  "extractMarkdown": false,
  "chunkMarkdown": false,
  "ingestVectors": false
}
```

3. **202 Accepted** = background job started. **409** = job already running for that `parentPageId`.
4. Poll **GET** `/api/confluence/ingest/status/{parentPageId}` until progress shows `COMPLETED`.

### Try query in Swagger

```json
{
  "query": "how do I configure SSL",
  "parentPageId": "12345",
  "topK": 5,
  "similarityThreshold": 0.0
}
```

No PAT required. **503** when query disabled or Chroma/Ollama unavailable.

### DTO reference (request bodies)

**`IngestionRequest`** — see [api.md](api.md). Key flags: `forceRebuildManifest`, `extractMarkdown`, `chunkMarkdown`, `ingestVectors`. Optional: `batchSize` (1–5000), `concurrency` (1–32), `requestTimeoutSeconds` (10–900).

**`QueryRequest`** — `query` (required), optional `parentPageId`, `topK` (1–50), `similarityThreshold` (0–1).

## Logging

### Configuration (`application.yml`)

| Setting | Value |
|---------|-------|
| Log file | `logs/app.log` |
| Root level | `INFO` |
| App package | `com.confluence.ingestor: INFO` |
| Pattern | `%5p [app-name,traceId,spanId]` — **traceId/spanId** injected by Micrometer tracing |

Example line:

```
 INFO [confluence-rag-ingestor,a1b2c3d4e5f6g7h8,i9j0k1l2m3n4] Background page transform started for parentPageId=12345 ...
```

### Enable DEBUG (local troubleshooting)

Add to `application-local.yml` or pass at startup:

```yaml
logging:
  level:
    com.confluence.ingestor.service: DEBUG
    com.confluence.ingestor.confluence: DEBUG
    com.confluence.ingestor.messaging: DEBUG
    com.confluence.ingestor.storage: DEBUG
```

Or single class:

```yaml
logging:
  level:
    com.confluence.ingestor.service.IngestionService: DEBUG
```

Run with profile:

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

`application-local.yml` also sets `default-concurrency: 1` and `per-page-state-enabled: true` for Windows-friendly runs.

### Security — never logged

- **PAT** (`IngestionRequest.pat`) — compact constructor explicitly avoids logging it.
- Page **body content** is not logged at INFO.
- Errors truncate to **2000 chars** when written to manifest/progress `error` fields.

### Log messages by pipeline stage

Use these grep patterns on `logs/app.log`:

| Stage | Logger / pattern | Level | Meaning |
|-------|------------------|-------|---------|
| Request options | `IngestionService` — `Ingestion options parentPageId=` | INFO | Resolved batch flags (visible at default log level) |
| API ingest | `IngestionController` — `Ingest request` / `Ingest response` | INFO | HTTP entry/exit |
| API query | `QueryController` — `Query request` / `Query response` | INFO | Query entry/exit |
| Job mutex | `IngestionJobCoordinator` — `rejected` / `already running` | INFO | 409 conflict reason |
| Confluence HTTP | `ConfluenceClient` — `GET content` / `list children` | DEBUG | Enable `com.confluence.ingestor.confluence: DEBUG` |
| Attachments | `AttachmentClient` — `Listed` / `Downloaded` | DEBUG | Byte counts only |
| Chroma ingest | `ChromaIngestionService` — `Chroma ingest` | INFO | Chunk upsert counts |
| Chroma query | `ChromaQueryService` — `Chroma search` | DEBUG | Hit counts |
| Markdown chunk | `MarkdownChunker` — `Chunked markdown` | DEBUG | Per-page chunk count |
| Validation | `GlobalExceptionHandler` — `validation failed` | WARN | Bad request bodies |
| Job dispatch | `InProcessJobPublisher` — `Publishing in-process job` | DEBUG | Which `IngestionJobType` fired |
| Kafka publish | `KafkaJobPublisher` — `Published` | INFO | When Kafka enabled |
| Manifest crawl start/end | `ManifestCrawlService` — `Background manifest crawl` | INFO | Crawl lifecycle |
| Crawl progress | `CrawlProgressService` — `Manifest crawl progress` | INFO | Every 100 processed pages |
| Crawl failure | `ManifestCrawlService` — `crawl failed` / `I/O error` | WARN/ERROR | Also in `crawl-progress.json` |
| Page transform batch | `PageTransformBatchService` — `page transform started/finished` | INFO | Includes batchSize, concurrency, counts |
| Single page transform | `PageTransformService` — `Markdown extraction` | INFO/WARN | Per-page success/failure |
| Chunk batch | `ChunkBatchService` — `chunk batch` | INFO | Same pattern as transform |
| Vector ingest | `VectorIngestService` — `Vector ingest complete/failed` | INFO/WARN | Per-page Chroma upsert |
| Vector batch | `VectorIngestBatchService` — `vector ingest batch` | INFO | Batch summary |
| Query | `QueryService` — `Query completed` / `Query failed` | INFO/WARN | hit count or error |
| Chained stages | `Chained page transform/chunk/vector ingest scheduled` | INFO | Auto-pipeline handoff |
| Skipped (409-like) | `skipped — no manifest` / `Skipped chained` / `duplicate` | WARN | Mutex or missing prereq |
| Atomic I/O retry | `FileStorageService` — `atomic replace` | WARN | Windows file lock |
| Batch progress write fail | `BatchProgressService` — `Failed to write batch progress` | WARN | Disk issue |

### On-disk progress (complement to logs)

| File | When updated | Key fields |
|------|--------------|------------|
| `crawl-progress.json` | Manifest crawl | `status`, `processedPages`, `totalPagesDiscovered`, `currentPageId`, `error` |
| `batch-progress.json` | Transform / chunk / vector batches | `phase`, `status`, `processedCount`, `failedCount`, `currentPageId`, `error` |
| `manifest.json` | After each page | `lastError`, `noOfRetries` per page |

Status API aggregates all of the above — prefer it over tailing logs for coarse progress.

## Distributed tracing

### Stack

| Component | Dependency / config |
|-----------|---------------------|
| Micrometer Observation | `spring-boot-starter-actuator` |
| OTel bridge | `micrometer-tracing-bridge-otel` |
| OTLP export | `opentelemetry-exporter-otlp` |
| AOP spans | `TracingConfig` → `ObservedAspect` bean |
| HTTP auto-instrumentation | Spring Boot MVC (incoming requests) |

### Configuration (`application.yml`)

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  tracing:
    sampling:
      probability: 1.0          # 100% of traces (reduce in prod)
  otlp:
    tracing:
      endpoint: http://localhost:4318/v1/traces
```

### Correlate logs ↔ traces

Log pattern includes `%X{traceId}` and `%X{spanId}`. When viewing traces in Jaeger/Zipkin/Grafana Tempo, search by **traceId** from any log line for that request.

### Local trace collector setup

OTLP HTTP endpoint defaults to `localhost:4318`. Options:

**Jaeger (all-in-one):**

```bash
docker run -d --name jaeger -p 16686:16686 -p 4318:4318 jaegertracing/all-in-one:latest
```

UI: http://localhost:16686 — service name `confluence-rag-ingestor`.

**Disable export** (logs only, no collector):

```yaml
management:
  tracing:
    export:
      enabled: false
```

Traces still populate `traceId` in logs via Micrometer context.

### Actuator endpoints for ops

| Endpoint | Purpose |
|----------|---------|
| `/actuator/health` | Liveness — component health |

## Common issues — what to check

| Symptom | Logs | Files / API | Trace |
|---------|------|-------------|-------|
| **409 ALREADY_RUNNING** | `Skipped chained` or coordinator acquire fail | `activeJob` in status API | Single long-running span |
| **Crawl stuck** | `Manifest crawl progress` stops increasing | `crawl-progress.json` `currentPageId` | Confluence HTTP errors in logs |
| **Pages skipped** | `Markdown extraction failed` | manifest `noOfRetries >= 2` | `page.transform` span errors |
| **Vector ingest fails** | `Vector ingest failed` | Chroma/Ollama running? | Check Spring AI connection errors |
| **Query 503** | `Query failed` / `Vector store not configured` | `query-enabled`, Chroma up | N/A |
| **Slow on Windows** | `FileStorageService` atomic replace warnings | Lower concurrency; use `local` profile | N/A |
| **Kafka duplicate skip** | `Skipped duplicate` in `KafkaIngestionJobConsumers` | `kafka.enabled=true` | Consumer group lag |
